web-agent

Python-only backend library and SDK wrapper.

I built this to show that web search is not magic. The useful core is small: web_search gets URLs, open_url retrieves the pages, and the LLM does the reasoning over the evidence.

GitHub PyPI 0.3.1 Internal runtime: web_agent_backend

Why I Built This

I wanted to prove that internet-grounded answers do not require a mysterious giant search stack. Two well-designed tools and a competent model already get you very far.

I also wanted a version that is materially cheaper than the common bundled research APIs people usually adopt first.

Usage

from langchain_openai import ChatOpenAI

from web_agent_sdk import WebAgentClient

llm = ChatOpenAI(
    model="gpt-5-nano",
    api_key="your-openai-key",
)

client = WebAgentClient(chat_model=llm)

quick = client.quick_search("Find pricing")
agentic = client.agentic_search("Investigate this company")

Architecture

The runtime flow is vertical and simple: ChatOpenAI and WebAgentClient, then web_search, then open_url, then LLM synthesis over the fetched evidence.

The Two Tools

web_search is the shortlist step. It uses Serper to return ranked URLs, snippets, and metadata.

open_url is the retrieval step. It fetches one page or a small batch, extracts readable text and markdown, and gives the model actual source material to reason over.

How The Crawler Works

The crawler validates the URL, tries HTTP first, retries transient failures, follows redirects, validates content type, extracts text, and only escalates to browser fetches when the domain profile or failure pattern requires it.
